## 一、OpenClaw 是什么

OpenClaw 是一个开源的个人 AI 智能体（Agent）框架，旨在让你在任何操作系统、任何平台上构建属于自己的 AI 助手。

### 一句话定义
**OpenClaw 是一个拥有长期记忆、可扩展技能、且能感知环境的自主 Agent 操作系统。**

### 和 ChatGPT/Claude/Cursor 的区别
| 特性 | ChatGPT/Claude | Cursor | OpenClaw |
|------|----------------|--------|----------|
| **本质** | 对话机器人 | AI 增强 IDE | **自主 Agent 平台** |
| **记忆** | 短期/Session级 | 项目代码级 | **三层长期记忆 (Tier 1-3)** |
| **能力** | 仅限对话/插件 | 仅限编程 | **全系统 Skills + MCP** |
| **运行** | 云端 | 本地 + 云端 | **本地优先 (Privacy First)** |

### 核心架构
- **Agent**: 核心决策大脑，基于 LLM（Claude/Gemini/DeepSeek 等）。
- **Skills**: 专项能力包，类似于"岗前培训"。
- **Memory**: 跨 Session 的持久化记忆体系。
- **MCP**: 外部服务标准接口（Model Context Protocol）。
- **Hooks**: 事件驱动的自动化逻辑。

## 二、部署与安装

### 三种方案对比
| 方案 | 适合人群 | 成本 | 复杂度 |
|------|---------|------|--------|
| **OneClaw (桌面版)** | 零代码新手 | 仅 API 费 | 极低 |
| **云端托管** | 不想维护服务器 | 约 199 元/月 | 低 |
| **Docker 自建** | 开发者/极客 | 约 2 美元/月起 | 中高 |

### Mac 安装 (推荐)
```bash
curl -fsSL https://openclaw.ai/install.sh | sh
```

### Windows WSL 安装
1. 安装 WSL2 (Ubuntu)。
2. 在 Ubuntu 终端中执行上述安装脚本。
3. ⚡ **实战建议**：建议使用 WSL 模式，因为 Linux 环境下 Skills 的兼容性最好。

## 三、核心概念

### Agent 文件体系
OpenClaw 的行为由工作区根目录的四个文件定义：
- `AGENTS.md`: 定义 Agent 的工作态度和通用规范。
- `SOUL.md`: 定义 Agent 的人格、语气和偏好。
- `USER.md`: 记录你的个人信息、习惯和时区。
- `MEMORY.md`: 存放最核心的长期记忆（控制在 100 行内）。

### 三层记忆模型
| 层级 | 名称 | 存储位置 | 特点 |
|------|------|----------|------|
| **Tier 1** | 信息层 | `memory/learning/` | 原始记录，只追加不删除 |
| **Tier 2** | 知识层 | `memory/YYYY-MM-DD.md` | 每日总结，提炼价值 |
| **Tier 3** | 智慧层 | `MEMORY.md` | 核心洞察，每次启动必加载 |

### Skills 系统
Skills 是 OpenClaw 的灵魂。它们是放在 `skills/` 下的 `SKILL.md` 文件。
- **怎么装**：`openclaw skills install <name>`
- **怎么写**：使用 `/skill-creator` 指令引导生成。
- **怎么迭代**：在 `AGENTS.md` 中要求 Agent 每次遇到错误都更新相关 Skill。

## 四、Skills 精选与推荐

### 新手必装 Top 5
1. **Capability Evolver**: 让 Agent 自我优化，总结使用习惯。
2. **Agent Browser**: 真的会操作浏览器的能力。
3. **Summarize**: 网页、PDF、长视频一键总结。
4. **Find Skills**: 帮你寻找和安装新技能。
5. **Clawsec**: **安装任何 Skill 之前先过一遍安全扫描**（来源：@DtDt666）。

### 进阶推荐
- **Tavily + Multi Search**: 实时搜索增强，同时跑 17 个引擎。
- **Boomerang Tasks**: 专门解决长任务失忆问题，自动拆分记忆。
- **Sequential Thinking**: 强制 Agent 在输出前进行深度推理。

## 五、记忆与知识管理

### MEMORY.md 的正确写法
不要把它当成垃圾桶。它应该是你人生的"索引"：
- 关键决策（为什么选 A 不选 B）。
- 核心偏好（我讨厌被催促，我喜欢简洁）。
- 重要的社交关系。

### Mem9 云端记忆层
由 TiDB 联创 Ed Huang 开发，解决多设备同步问题。基于 TiDB Cloud，提供 25GB 免费额度。
- **安装**：`openclaw plugins install @mem9/mem9`

## 七、进阶玩法

### Harness Engineering (控制论思维)
> "Agent 失败不是因为能力不足，而是你没有把'什么是好的'外化出来。" —— @odysseus0z

核心在于把你的隐性判断变成机器可读的：
- 编写 Golden Principles 文档。
- 配置自定义 Linter。
- 构建闭环的反馈传感器。

### OpenCLI 浏览器自动化
比 Playwright MCP 更轻量，复用你 Chrome 浏览器的登录态，解决一切 Cookie 难题。

## 九、常见问题 FAQ

**Q: Skills 装太多会怎样？**
A: 会占用大量 Context Window (上下文窗口)，导致启动变慢，建议只保留 5-8 个核心 Skill。

**Q: Token 消耗太快怎么办？**
A: 使用异构模型策略。简单任务用 DeepSeek/Gemini Flash，核心决策才调 Opus/GPT-4。
